# SanWich v2.3.2 更新紀錄

## 本版重點

- 改善繁體中文字幕的語意斷句，降低把連接詞、修飾語、數字單位與英文片語拆開的情況。
- 改善中英數空格與數字標點處理，保留小數、時間、版本號、千分位與數字範圍。
- 修正 Breeze 每 60 秒分段時可能產生的字幕時間碼重疊。
- AI 校對改為記錄實際完成範圍：
  - 橘色：AI 有修改。
  - 綠色：AI 已檢查但未修改。
  - 灰色：該段 AI 未完成，已保留 Breeze 原文。
- 大型 SRT 編輯器改為分頁顯示，每頁最多 160 組，降低數千組字幕時的開啟與操作負擔。
- 時間軸只繪製目前可視範圍，改善長影片拖動與縮放效能。
- 時間碼檢查失敗時，會自動跳到包含錯誤字幕的頁面與列。
- 改善字幕插入／刪除後的修改紀錄對齊，避免後續大量字幕被誤判為人工修改。
- 修正 Free 模式仍可能在背景套用既有個人化規則的問題；個人化規則現在完整遵守 Supporter 功能邊界。
- 修正開發測試啟動檔搬移後找不到主程式的問題。

## 相容性與資料

- API 設定、個人化規則與授權資料仍保存在使用者資料夾，更新程式不會覆寫。
- 舊版輸出的 SRT、TXT 與個人化規則資料可以繼續使用。
- Free 核心功能維持不變：單檔轉寫、AI 校對、SRT／TXT 輸出與基本字幕編輯器仍可使用。

## 已知限制

- Breeze-ASR-25 首次使用仍需下載約 3–4 GB 模型。
- 語者分離目前只套用於 TXT，不會在 SRT 內加入語者名稱。
- AI 校對的品質與可用額度取決於使用者選擇的 API 供應商與模型。
