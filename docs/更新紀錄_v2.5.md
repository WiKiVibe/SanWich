# SanWich v2.5 更新紀錄

> 發佈日期：2026-07-23

## 本機私密 AI

- 新增 Breeze-7B 本機校對模式，不需要 API Key，文字只送到本機 `127.0.0.1`。
- ASR 批次完成後先釋放 Breeze-ASR-25 與 CUDA 快取，再啟動本機 LLM，降低低顯存顯卡同時載入失敗的機率。
- 首次下載支援 `.part` 續傳、SHA-256 驗證與磁碟空間預檢。

## 字幕播放器與校對流程

- 新增低延遲字幕播放器、波形快取與字幕區段定位。
- AI 固定 TC 斷句保留時間碼；局部失敗時只保留該窗口原結果，不再破壞其他已完成區段。
- 補強語意斷句、上下文拼字、批次完整性與字幕編輯歷史的回歸測試。

## 14 天試用與線上完整版授權

- 新使用者第一次啟動可使用 14 個曆日的完整版試用。
- 試用結束後自動回到基本功能；單檔辨識、校對、SRT／TXT 輸出與基本編輯器仍可使用。
- 舊使用者已存在的 Trial 到期日不縮短、不重設。
- 新版完整版 Key 透過 WiKiVibe License Server 啟用，預設最多 2 台裝置。
- 啟用後每 30 天重新驗證，另有 14 天離線寬限；明確撤銷、停用或到期會清除本機新版 Token。
- 設定頁新增「重新驗證」、「停用此裝置」及舊 `SW2` Key 明確遷移入口。
- API 或授權服務失敗不會鎖住基本功能。

## 程式內一鍵更新

- 啟動時自動檢查 GitHub Release，設定頁也可手動按「檢查更新」。
- 安裝版會在軟體內下載並驗證下一版 `SanWich_Setup_v版本.exe`，關閉舊版後靜默覆蓋安裝並重新啟動；模型、授權與個人設定不會重複下載或被覆寫。
- 更新前會核對 GitHub Release 提供的檔案大小與 SHA-256，驗證失敗不會執行安裝程式。
- 更新時只替換安裝目錄內的程式檔；模型、快取、API Key、個人規則與授權資料位於使用者資料夾，不會被清除。
- 開始更新前會暫存目前的程式檔；如果安裝程式回報失敗，會復原並重新啟動原版本。

## 儲存位置

| 資料 | 位置 |
| --- | --- |
| API Key 與設定 | `%APPDATA%\SanWich\config.json` |
| 個人化規則 | `%APPDATA%\SanWich\personal_rules.json` |
| Trial 狀態 | `%APPDATA%\SanWich\license.json`，另有 `%LOCALAPPDATA%` 與登錄檔錨點 |
| 新版授權 Token | `%APPDATA%\SanWich\license_v2.json` 與 `%LOCALAPPDATA%\SanWich\license_v2_anchor.json` |
| 本機 AI 與快取 | `%LOCALAPPDATA%\SanWich` |

## 驗證

- SanWich 單元測試、GUI smoke、更新包套用與資料保留測試通過。
- 正式 License Server 已完成兩台啟用、第三台拒絕、停用釋放名額、重新驗證及撤銷拒絕的線上端到端測試。
- 正式發佈提供單一 `SanWich_Setup_v2.5.exe` 安裝包；不包含使用者 API Key、模型或個人資料。
