# SanWich v2.5b 更新紀錄

> v2.5 正式發佈前的 beta 命名。  
> 主題：**本地私密 AI＋低延遲 SRT 預覽播放器**。

## 本地私密 AI

- 新增「本機私密 AI」供應商，不需要 API Key。
- 以 Breeze-7B-Instruct v1.0 Q4_K_M 在本機校對台灣繁體中文字幕。
- llama.cpp 只監聽 `127.0.0.1`，字幕與 Prompt 不送往第三方服務。
- 首次使用可從設定頁下載模型及執行核心，支援續傳與 SHA256 驗證。
- RTX 50 系列自動選 CUDA 13；RTX 20／30／40 系列選 CUDA 12；無 NVIDIA GPU 時使用 CPU runtime。
- 依 VRAM 自動調整 context 與 GPU layers，GPU 啟動失敗時保留 CPU 備援。
- 自動尋找 8080 之後的空閒連接埠，避免和其他本機服務衝突。
- 本機模式改為兩階段處理：先完成全部 Breeze-ASR，再釋放 PyTorch／CUDA 顯存，最後啟動本機 LLM 校對；避免 6GB 顯卡同時載入兩個模型。
- 首次下載前檢查本機 AI 所在磁碟；低於 7GB 直接停止下載，7～10GB 顯示空間警告。

## SRT 編輯器

- 保留 FFmpeg 建立 PCM WAV 代理檔，但主要播放改為常駐 PortAudio 串流。
- 播放游標使用音訊裝置的 DAC 時鐘，修正舊 HDD 電腦上波形先走、聲音後到的問題。
- 播放、暫停、seek 與拖曳預聽不再反覆啟動 `ffplay.exe`。
- 波形代理檔加入快取；重開相同媒體不必重新轉換。
- 長音訊波形採分段讀取，降低記憶體尖峰。
- PortAudio 無法使用時仍可回退 `ffplay`，並留下診斷日誌。

## 安裝與啟動

- 安裝器新增 `sounddevice`。
- 捷徑透過 `run_hidden.vbs` 隱形啟動，錯誤仍寫入 `logs`。
- 版本號更新為 `2.5b`；下一次正式公開發佈使用 `v2.5`。

## 本次實機驗證

- RTX 3060 12GB：自動選擇 CUDA 12.4 runtime，Breeze-7B 以 GPU layers 99 啟動成功。
- 4.54GB GGUF 完整下載、續傳路徑與 SHA256 驗證通過。
- 空白 API Key 的 `local` 供應商成功走完 `CORE.llm_merge`，單組測試字幕約 0.78 秒完成。
- 8080 被占用時，自動改用 `127.0.0.1:8081`；測試結束後程序與連接埠正常釋放。
- Windows WASAPI 實際開啟成功；16kHz 單聲道 PCM 使用 256-frame block，播放器位置由 DAC clock 推進。
- 既有與新增單元測試共 47 項通過，包含 `2.5b` 可正確收到正式 `2.5` 更新提示、低顯存模型交接順序與磁碟空間預檢。

## 尚待實機驗證

- 舊電腦搭配 HDD 的 SRT 播放延遲、波形與聲音同步。
- 全新電腦、無既有快取的乾淨安裝。
