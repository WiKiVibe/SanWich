## 3. 分檔詳細修改

### 3.1 `local_llm.py`（本機 GPU／runtime）

**關鍵函式／符號（搜尋用）：**

- `driver_cuda_version`
- `runtime_variant`（依驅動 + GPU 選套件）
- `reinstall_runtime`
- `_launch_env`（PATH 把 llama 目錄放最前，避開系統 CUDA Toolkit）
- `_detect_cuda_failure` / `_report_runtime_mode`
- `ensure_running` 內「已裝版本 ≠ 需要版本 → 自動重裝」

**`runtime_variant` 規則（重要）：**

```text
有 NVIDIA：
  驅動 CUDA >= 13.0 且 RTX 50 / Blackwell → cuda-13.3
  否則（含 RTX 5070 + 驅動 CUDA 12.9）→ cuda-12.4
無 NVIDIA → cpu
```

**啟動行為：**

1. 讀 `nvidia-smi` 的 `CUDA Version:`  
2. 若 manifest 是 13.3 但需要 12.4 → 清 runtime 重下  
3. 啟動時 `env PATH = llama.cpp;...`（去掉 Toolkit 路徑）  
4. 健康檢查後掃 log：若仍有 CUDA fail → 明確警告「像在跑 CPU」  
5. 若先前是 CPU／CUDA fail 的實例 → 不沿用，強制重啟  

**工作機已驗證：**

```text
cuda-13.3 → 重裝 cuda-12.4 後
GPU 模式就緒 layers=99
生成約 110 t/s（先前 CPU 約 10 t/s）
```

### 3.2 `SanWich.py`（設定 UI + 流程）

#### （1）供應商／Key／模型記憶

**關鍵符號：**

- `PROVIDER_ORDER`
- `ensure_provider_memory`
- `_guess_key_home_provider`（`AQ.` / `AIza` → gemini；`sk-ant` → claude）
- 設定窗：`apply_settings_to_cfg`、`on_provider_change`、`close_and_save`

**config 新增欄位（寫入 `%APPDATA%\SanWich\config.json`）：**

```json
{
  "api_provider": "deepseek",
  "model": "deepseek-v4-flash",
  "api_key": "（目前供應商的 key，相容舊欄位）",
  "api_keys_by_provider": {
    "local": "",
    "gemini": "...",
    "openai": "...",
    "claude": "...",
    "deepseek": "..."
  },
  "models_by_provider": {
    "local": "Breeze-7B-...",
    "gemini": "gemini-2.5-flash",
    "openai": "...",
    "claude": "...",
    "deepseek": "deepseek-v4-flash"
  }
}
```

**行為：**

- 切換供應商 → 立刻存檔 + 換上該家 Key／上次模型  
- 關窗／右上角 X → 也會存（不再等於取消）  
- 重開顯示上次供應商  

#### （2）本機 AI 流程

- 本機 `max_tokens`: **2048 → 4096**（`_llm_call_once_with_deepseek` local 分支）  
- `provider == "local"` 時 **略過 AI 語意斷句**（`skip_ai_semantic`），改本機安全斷句（省時間、少失敗）  

#### （3）文檔匹配入口

- `apply_script_document_notes(...)`  
- 未開 AI、或 AI 失敗時，仍可用完整腳本做對齊覆寫  

#### （4）UI 文案

- 補充資料 hint／placeholder 改為：整份腳本（文檔匹配）或短詞庫／`錯 > 對`

### 3.3 `SanWich_legacy_core.py`（校對核心）

#### （1）補充資料／詞庫

**新增／強化：**

| 符號 | 用途 |
|------|------|
| `SRT_WITH_GLOSSARY_PROMPT` | 有補充、未開總編輯時用（不再「盡量原樣」） |
| `SUPPLEMENT_GLOSSARY_RULES` | 詞庫優先規則 |
| `replacements_from_context` | `>` `→` `=` `：` `改成` 等 |
| `required_phrases_from_context` | 一行一正確寫法 |
| `canonical_terms_from_context` | 英數專名（含多詞） |
| `apply_context_replacements` | 硬替換 |
| `apply_context_canonical_terms` | 英文大小寫硬套 |
| `apply_context_required_phrases` | 正確寫法套用 |
| `_fuzzy_replace_cjk_phrase` | 中文近似（同長度差 1 字等） |

**`llm_merge` 後處理順序：**

1. （若文檔模式）`document_match_texts`  
2. `錯 > 對` 硬替換  
3. 短詞庫／中文近似  
4. 英數專名大小寫  

#### （2）文檔／腳本匹配（類似剪映）

**關鍵符號：**

- `is_script_document`
- `extract_script_document`
- `document_match_texts`
- `_best_script_window`

**判定為「腳本」的條件（約）：**

- 行首有 `【腳本】`／`【文檔】`／`逐字稿`… 標記，或  
- 去空白後 ≥ 約 45 字，或  
- 單行夠長／多行連貫口播  

**演算法概要：**

- 腳本與字幕去標點、去空白做字元流  
- 依序為每句在腳本剩餘區找最像窗格  
- 相似度夠 → 用腳本對應原文覆寫（時間碼不動）  

**使用方式（給使用者／測試）：**

```text
【腳本】
完整口播逐字稿貼在這裡……

或短詞庫：
葉黃素
示範頻道
夜黃素 > 葉黃素
```

#### （3）語意斷句（較早的小改）

- Prompt 寫清：可改切點、不可改字  
- 失敗 log 標類型（改字／JSON／截斷…），不再一律「偷改文字」  
- 本機流程上已多半略過此步（見 SanWich.py）
